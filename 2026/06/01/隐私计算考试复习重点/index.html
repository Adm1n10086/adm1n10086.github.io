# 第一章 绪论

课堂提问：数据采集、数据传输、数据存储、数据分析各个阶段有什么样的一个问题
作业
总结大数据生命周期存在的安全威胁以及对应的解决技术
大数据的生命周期包括数据产生、采集、传输、存储、使用、分享、销毁等诸多环节，每个环节都面临不同的安全威胁。

测试
请论述数据采集阶段涉及的2种大数据安全技术的基本原理及可能的应用场景

大数据生命周期

一、数据采集阶段
（1）2 种大数据安全技术
①安全多方计算
原理：多个数据持有方在不泄露各自原始采集数据的前提下，协同完成数据统计、模型运算，通过密码学协议拆分计算逻辑，各参与方仅持有私密分片数据。
应用场景：多机构用户行为数据联合采集统计，如多家金融机构在本地采集用户信贷数据，不出原始数据联合风控建模。
②可验证计算
原理：终端采集数据后外包运算，计算方输出结果附带正确性证明，采集方可低成本核验运算未被篡改，保障预处理运算可信。
应用场景：物联网传感器采集海量环境数据后，云端预处理校验，防止服务商恶意篡改采集原始数据。
（2）2 种大数据隐私保护技术
①本地差分隐私 (LDP)
原理：用户终端本地对采集数据添加可控随机噪声，扰动真实数据后再上传，攻击者无法从上传数据反推单用户真实信息。
应用场景：手机 APP 采集用户浏览、位置数据，本地加噪后上报服务商，保护个体用户隐私。
②属性匿名
原理：对采集数据集里用户敏感属性（姓名、身份证）做泛化、隐匿处理，删除或模糊精准标识字段，切断属性与自然人关联。
应用场景：医疗平台采集病患诊疗数据，脱敏隐匿患者身份属性，用于医疗大数据汇总分析。
二、数据传输阶段
（1）2 种大数据安全技术
①SSL 加密协议
原理：基于非对称密钥协商 + 对称加密，建立传输加密通道，握手阶段协商会话密钥，传输数据全程密文传输，附带消息验证码校验完整性。
应用场景：浏览器与服务器间用户表单、设备采集数据互联网传输，日常 APP 接口数据交互。
②VPN 技术
原理：在公网链路构建加密虚拟私有隧道，数据包封装加密后隧道传输，隔离公网窃听链路。
应用场景：企业分支机构物联网终端采集数据跨地域向总部数据中心内网传输。
（2）2 种大数据隐私保护技术
①身份匿名
原理：传输报文剔除用户真实身份标识，替换为临时匿名编号，链路截获报文无法定位真实用户。
应用场景：网约车终端位置数据回传平台，传输报文隐藏司机、乘客真实 ID。
②节点匿名（社交图谱隐私）
原理：传输社交拓扑数据时，模糊图谱节点主体标识，隐藏节点对应的自然人信息。
应用场景：社交软件用户关系链数据跨服务器同步传输，规避链路窃听扒取用户社交关系。
三、数据存储阶段
（1）2 种大数据安全技术
①基于属性加密 (ABE) 的访问控制
原理：密文绑定访问属性，用户密钥匹配预设属性才可解密读取存储数据，属性不匹配无法访问数据。
应用场景：企业云端存储员工人事数据，不同部门管理员凭对应属性密钥访问分级存储数据。
②风险自适应访问控制
原理：实时评估访问主体、访问环境风险等级，动态调整数据读写权限，高风险访问行为自动拦截。
应用场景：政务大数据中心存储公民政务数据，异地陌生设备访问自动收紧数据读取权限。
（2）2 种大数据隐私保护技术
②多次发布匿名技术
原理：数据集多次入库存储发布时，迭代调整匿名规则，规避攻击者利用多版数据交叉比对还原用户隐私。
应用场景：人口统计数据库分季度迭代存储更新，反复匿名处理防止差分溯源用户信息。
③位置轨迹匿名
原理：存储用户轨迹数据时，模糊精准坐标、合并相邻轨迹点位，打散用户连续行踪链路。
应用场景：地图服务商云端存储用户出行轨迹，匿名后归档存储防范轨迹泄露。
四、数据分析与使用阶段
（1）2 种大数据安全技术
①同态加密
原理：密文状态下直接开展加减乘等数据分析运算，运算结果解密等价于明文计算结果，分析全程不还原原始明文。
应用场景：服务商接收加密用户数据，在密文上完成用户消费画像统计，无法查看原始消费明细。
②可搜索加密 (SSE)
原理：数据加密存储后，依托陷门密钥实现密文关键词检索，检索过程不泄露明文内容，安全检索后开展数据分析。
应用场景：律所海量加密卷宗数据，密文检索关键词调取案卷用于案件大数据分析。
（2）2 种大数据隐私保护技术
①全局差分隐私
原理：数据集整体统计结果添加标准化噪声，任意单条数据增删不会显著改变统计输出，阻止通过统计差值反推个体隐私。
应用场景：电商平台依托用户订单数据做销量统计分析，差分隐私保护单个用户购买记录。
②联邦学习
原理：各数据方保留本地原始数据，仅上传模型参数更新值协同迭代训练，原始数据不出本地完成联合数据分析。
应用场景：多家银行联合训练信贷风控模型，各自储户数据留存在本地，仅交互模型梯度参数

# 强制访问控制模型：BLP Biba

图2.3

BLP（机密性）：
基本概念：
安全级别Level：公开（UC）、秘密（S）、机密（C）、绝密（TS）。它们之间的关系为UC≤S≤C≤TS。
范畴Category：为一个类别信息构成的集合，例如｛中国，军事，科技｝。具有该范畴的主体能够访问那些以该范畴子集为范畴的客体。
安全标记Label：由安全级别和范畴构成的二元组<Level，Category>，例如<C,｛中国,科技｝>。
支配关系dom：安全标记 A dom B，当且仅当Level_A≥Level_B，Category_A⊇Category_B。

访问行为应满足如下两条安全属性
简单安全属性：主体S可以读客体O，当且仅当LabelS dom LabelO，且S对O有自主型读访问权限。
*安全属性：主体S可以写客体O，当且仅当LabelO dom LabelS，且S对O具有自主型写权限。

Biba（完整性）：
基本概念：
完整性级别Level：代表了主/客体的可信度。完整性级别高的主体比完整性级别低的主体在行为上具有更高的可靠性；完整性级别高的客体比完整性级别低的客体所承载的信息更加精确和可靠。
范畴Category：若范畴Category_A⊇Category_B，则A能写入B；否则，A不能写入B。
完整性标记Label：由完整性级别和范畴构成的二元组<Level，Category>。
支配关系dom：完整性标记A dom B，当且仅当Level_A≥Level_B，Category_A⊇Category_B。

安全属性：
完整性特性：主体S能够写入客体O，当且仅当LabelS dom LabelO。
调用特性：主体S1能够调用主体S2，当且仅当LabelS1 dom LabelS2。（与BLP不同）
简单完整性条件：主体S能够读取客体O，当且仅当LabelO dom LabelS。

区别与联系：
区别

维度 BLP 模型 Biba 模型
核心目标 数据机密性 数据完整性
控制方向 防止高密级信息向下泄露 防止低完整性数据向上污染

# 描述凝聚式角色挖掘的基本思想，基于凝聚式层次聚类技术对表2-1进行分析。

表2-1：

基本思想：
它将权限看作是聚类的对象，初始将每个权限作为一个类簇，通过不断**自底向上(用户->角色)**地合并距离近的类簇完成对权限的层次聚类，聚类结果为候选的角色及它们的继承关系。
注：两个权限类簇之间的距离是由它们之间的共同用户数量以及它们所包含的权限数量决定的，两个类簇的共同用户数量越多，且包含的权限数量越多，则两个类簇的距离越近。
大致步骤：
为所有单个权限r创建一个类簇cr。合并距离相近的类簇对产生新类簇，距离最近的类簇对的寻找方式为先寻找出拥有共同用户最多的类簇对集合S，再从S中选出包含权限最多的类簇对集合E。

# 描述风险自适应访问控制技术的基本思想。

基本思想
风险自适应访问控制技术的基本思想是，将访问控制中隐含的风险概念明确化，根据访历史问行为带来的风险，动态地赋予访问权限，在访问过程中动态实施风险与收益的权衡，而非预先在静态策略中定义。

# 描述基于单发送者的广播加密的访问控制方案。

参与方：数据所有者持有完整密钥树，按需选密钥加密并全网广播密文；普通用户仅保有自身叶子密钥与上行祖先密钥，接收密文尝试解密。
密钥树：用户密钥树中的所有密钥均为对称密钥，叶子节点绑定单个用户密钥，非叶子节点为对应子树用户组密钥，用户持有从自身叶子到根的路径密钥。
加解密与权限：发送方挑选若干组密钥，各密钥单独加密明文生成多份密文广播；授权用户自有密钥匹配加密密钥之一可解密，未授权用户无匹配密钥无法解密，以此完成访问管控。
注：基于密码学的访问控制技术的安全性依赖于密钥

# 描述基于公钥广播加密的访问控制方案。

以公钥服务器为核心，适配多发送者场景。
公钥服务器为用户子集生成公私钥对并分发私钥；
数据所有者用对称密钥加密数据，再用授权用户公钥加密对称密钥，生成密钥块并入元数据；
授权用户凭借自身私钥解密得到对称密钥以访问数据，可实现读写权限分离，摆脱了单发送者方案的约束。

# 描述对称密文检索和非对称密文检索方案并对比分析两者的区别和联系。

根据应用场景的不同，密文检索技术可以分为对称密文检索SSE和非对称密文检索ASE两大类：
对称密文检索：在对称密钥环境下，只有数据所有者拥有密钥，也只有数据所有者可以提交敏感数据、生成陷门，即数据所有者和数据检索者为同一人。
非对称密文检索：在非对称密钥环境下，任何可以获得数据检索者公钥的用户都可以提交敏感数据，但只有拥有私钥的数据检索者可以生成陷门。

特性 对称密文检索 非对称密文检索
密文和索引的构建 由私钥生成 由公开参数生成
密钥管理 单用户场景 多用户场景
性能 高效 低效
解决的问题 不可信的服务器存储 不可信的服务器路由
注
  • 系统模型：
    密文检索主要涉及数据所有者、数据检索者以及服务器3种角色。
    数据所有者是敏感数据的拥有者（具备有限的存储空间和计算能力）
    数据检索者是查询请求的发起者（具备有限的存储空间和计算能力）
    服务器为所有者和检索者提供数据存储和数据查询服务，其由云存储服务提供商进行管理和维护，并具有强大的存储能力和计算能力。
  • 系统流程：
    数据所有者首先为数据构造支持检索功能的索引，同时使用传统的加密技术加密全部数据，然后将密文数据和索引共同存储到服务器。
    需要检索时，数据检索者为检索条件生成相应的陷门，并发送给服务器。
    服务器使用索引和陷门进行协议预设的运算，并将满足检索条件的密文数据返回给数据检索者。
    数据检索者使用密钥将检索结果解密，得到明文数据。有时服务器返回的密文数据中可能包含不满足检索条件的冗余数据，此时数据检索者还需要对解密后的明文数据进行二次检索，即在本地剔除冗余数据。
  • 安全模型:
    安全模型：
    通常认为数据所有者和数据检索者是完全可信的
    而服务器属于攻击者，其对用户的敏感数据和检索条件比较好奇
    目前大部分密文检索方案均假设服务器是诚实而好奇的（Honest-but-Curious，HBC），即服务器会忠实地执行数据检索者提交的检索请求，并返回相应的检索结果，同时其可能会利用自己所掌握的一切背景知识来进行分析，期望获得真实的敏感数据和检索条件。
  • 检索性能:
    检索性能：
    密文检索方案的性能主要从3个方面进行考虑:
    数据所有者的索引生成效率;
    数据检索者的陷门生成效率;
    服务器的检索效率。（关键！）
    由于索引的生成过程是一次性的,而陷门则是数据检索者根据自己的检索条件构造的,消耗时间一般较少。
    因此，本章主要关注检索效率,即服务器使用陷门和索引完成查询操作的时间。

# 描述文档-关键词索引方案、关键词-文档索引方案的基本思想，并比较两个方案的区别和联系。

⭐两种方案的基本思想
文档 - 关键词索引方案
以文档为中心，为每篇文档单独构建一个独立索引（如基于布隆过滤器的加密索引）。服务器检索时，需要遍历所有文档的索引，逐一判断是否包含查询关键词，时间复杂度与文档总数成正比。
关键词 - 文档索引方案
以关键词为中心，为每个关键词构建一个索引结构（类似倒排索引），记录所有包含该关键词的文档标识集合。服务器检索时，直接定位到查询关键词对应的索引，无需遍历所有文档，时间复杂度仅与结果数量成正比。
⭐两种方案的区别与联系
区别

# 以基于对称的比特承诺方案为例说明比特承诺方案中的隐藏性和绑定性

承诺阶段：
Bob产生一个随机数R，将R给Alice
Alice将R和想承诺的位b连接，并用K加密，即S=(K,R||b),传给B
揭示阶段：
Alice将密钥K给B
B用K解密钥得到R'||b，验证R'=R，揭示承诺b
这里面密钥K使得Bob不能提前揭示承诺（隐藏性）（仅由S无法推出b的值）
随机串R使得Alice不能篡改承诺（绑定性）（改变b的值，必然改变S的值，导致R'≠R）
注：本题考查的是承诺，承诺有绑定性与隐藏性，封你为承诺阶段和揭示阶段

# 以交互式的Schonrr协议为例说明什么是零知识证明

前提，A公钥 pk=aG ，私钥sk=a，目的是让B知道A有私钥a，并且不泄露a
A先生成一个随机数r，映射到椭圆曲线上，记为R=rG
B生成一个挑战数字c，发给A
A生成z=r+ac=r+csk，发给B
B收到z=r+ac，检验zG？=R+cpk(两边加相同生成元)，如果相等，可验证A确实有私钥a
由于椭圆曲线的离散对数问题，由R=r*G无法推断出r，保护了r的私密性与sk的零知识性

注：零知识性证明，是指证明者向验证者证明自己有该秘密，并且证明过程中不泄露该秘密，有三个性质[完备性、可靠性、零知识性]

差分隐私

差分隐私的核心优势：
同时保障「安全性」和「数据可用性」
如果方案能够实现这一安全目标， 就能够达成两种效果:
首先，因为无论攻击目标是否在查询数据集中， 查询结果都基本保持不变， 所以攻击者无法根据查询结果确认攻击目标是否在查询数据集中，也就无法实现链接攻击。（安全性）
其次， 这一安全目标有效地保持了数据可用性。无论单个数据记录加入或离开数据集D，对这一数据集的查询结果都基本保持稳定，也可以说保持了数据中有用的知识。（可用性）

差分隐私模型的安全目标：
首先，攻击者无法确认攻击目标在数据集中。
其次，即使攻击者确认攻击目标在数据集中，攻击目标的单条数据记录对输出结果的影响并不显著， 攻击者无法通过观察输出结果获得关于攻击目标的确切信息。

# 什么是联邦学习

联邦学习为一种具有以下特征的机器学习算法框架:
**多方参与:**有两个或以上的参与方合作构建共享的机器学习模型。每一个参与方都拥有一部分数据用以训练模型;
不交换数据:在联邦学习训练过程中，任意一个参与方的任意原始数据不会离开该参与方，不会被直接交换和收集;
保护传输信息:在联邦学习训练过程中，训练所必需的信息需要经过保护后在各参与方之间进行传输,使得各参与方无法基于传输的信息推测其他参与方的数据;
近似无损:联邦学习模型的性能要充分接近理想模型(即各参与方通过直接合并数据训练得到的模型)的性能。
联邦学习可以理解为分布式学习的一种，带约束的分布式学习。

# 联邦学习的分类

横向联邦学习(特征对齐的联邦学习)
各个参与者的业务类型相似，数据特征重叠多，样本重叠少。例如：不同地区的两家银行。多家银行就都可以上传自己的模型参数，然后对参数进行更新，完成模型效果的提升。
纵向联邦学习 (样本对齐的联邦学习)
如果参与者的数据中，样本重叠多，特征重叠少，例如：同一地区的银行和电商，就需要先将样本对齐，其中由于不能直接进行信息匹配，可以借助加密算法，让参与者在不泄露信息的情况下，找出相同的样本后联合他们进行特征，构建一个大表，进行模型训练。
联邦迁移学习
如果样本重叠不多，特征重叠也不多，希望利用数据提升模型能力，就需要将参与者的模型和数据迁移到同一空间中运算

# 联邦平均聚合的基本思想

核心：在不收集用户原始数据的前提下，分布式训练全局模型。
初始阶段：服务端下发全局模型给所有参与的本地客户端。
本地训练：各客户端用本地私有数据独立训练，更新得到本地模型参数，不上传原始数据。
参数上传：客户端仅把模型参数传回服务端。
加权聚合：服务端对所有本地参数按样本数量加权平均，生成新一轮全局模型。
迭代循环：重复下发、本地训练、聚合，直到模型收敛。
简言之：数据留本地，只传模型参数，服务器做平均聚合，兼顾隐私与联合建模。
