# 第一章：

课堂提问：数据采集、数据传输、数据存储、数据分析各个阶段有什么样的一个问题
作业
总结大数据生命周期存在的安全威胁以及对应的解决技术
数据采集阶段
核心威胁
非法越权采集、恶意爬虫窃取数据、采集过程中用户身份 / 敏感属性泄露、数据采集不完整 / 被篡改。
对应防护技术
大数据访问控制（对应图中模块①）：
风险自适应访问控制、角色挖掘、基于密码学的访问控制，实现采集权限的精细化管控，防止非法采集。
隐私保护技术（模块④）：
关系数据匿名化（身份匿名、属性匿名），在采集时对用户身份、敏感字段做匿名处理，避免直接泄露。
数据传输阶段
核心威胁
传输窃听、中间人篡改、重放攻击、数据泄露。
对应防护技术
基础安全技术：TLS/SSL 加密、VPN、IPsec、数字签名与消息验证码。
结合框架：通过访问控制与加密技术，保障传输链路的可信与数据机密性。
数据存储阶段
核心威胁
非法访问、数据泄露、密文数据检索时的隐私泄露、存储介质数据残留。
对应防护技术
安全检索（模块②）：
可搜索加密（SSE/ASE）、密文区间检索、PIR/ORAM，实现密文数据的安全检索，避免明文泄露。
基础安全技术：全磁盘 / 文件加密、分级访问控制、数据备份与容灾。
数据分析 / 计算阶段
核心威胁
数据计算时的隐私泄露、第三方外包计算不可信、模型推理攻击、合谋攻击。
对应防护技术
安全计算（模块③）：
同态加密、函数加密、安全多方计算、外包计算 / 可验证计算，实现 “密文可计算”，计算过程不泄露明文。
隐私保护技术（模块④）：
差分隐私（基本差分隐私、本地差分隐私）、位置轨迹隐私保护，在分析过程中加入可控噪声，保护个体隐私；
社交图谱 / 关系数据匿名化，避免分析过程中通过关联推理还原用户信息。
