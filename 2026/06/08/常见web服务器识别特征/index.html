注：以下特征只是常见的特征，不是100%识别特征，这些特征极易伪造，不适用于红蓝攻防演练

# 🔐 安全/权限框架

1. Apache Shiro

  • 最典型特征：请求/响应中出现 rememberMe Cookie，响应头常返回 Set-Cookie: rememberMe=deleteMe
  • 路径特征：/login, /logout, /captcha
  • 漏洞关联：反序列化、弱密钥爆破

2. Spring Security

  • 路径特征：/login, /logout, /oauth2/, /sso/
  • 响应头：X-Content-Type-Options, X-Frame-Options 等安全头常见
  • Cookie：JSESSIONID + 自定义令牌（如 XSRF-TOKEN）
  • 状态码：302 重定向到登录页、401 Unauthorized

# 🚀 Java Web 框架

1. Spring Boot/Spring MVC

  • 特征文件：/actuator/ 系列端点（/actuator/health, /actuator/env）
  • 响应头：Server: Apache-Coyote, X-Application-Context
  • 错误页：Whitelabel Error Page 或带 error 路径的JSON错误信息
  • 常见路径：/swagger-ui.html, /v2/api-docs

2. Struts2

  • 特征参数：URL中带 .action、.do 后缀
  • 响应头：X-Powered-By: Servlet/2.5 (Tomcat/...)
  • 漏洞关联：/struts/ 路径、OGNL表达式注入特征

3. JSP

  • 特征：URL后缀 .jsp、.jspx
  • Cookie：JSESSIONID
  • 响应头：X-Powered-By: JSP/2.2

# 🧩 PHP 框架

1. ThinkPHP

  • 特征：URL路径如 /index.php?s=/、/index.php?m=xxx&c=xxx&a=xxx
  • 响应头：X-Powered-By: ThinkPHP
  • 错误页：暴露版本号的ThinkPHP错误信息
  • 漏洞关联：/public/ 目录、路由解析特征

2. Laravel

  • Cookie：laravel_session
  • 路径特征：/routes/web.php 相关路由，/storage/ 目录
  • 响应特征：JSON错误页、csrf-token 隐藏表单字段

3. CodeIgniter

  • URL特征：/index.php/controller/method
  • Cookie：ci_session

# 📄 .NET 框架

1. ASP.NET / ASP.NET MVC

  • 特征：URL后缀 .aspx、.ashx
  • Cookie：ASP.NET_SessionId
  • 响应头：X-Powered-By: ASP.NET
  • 错误页：Server Error in '/' Application

2. ASP.NET Core

  • 响应头：Server: Kestrel
  • 路径特征：/swagger/index.html
  • 错误页：An unhandled exception occurred

# 🐍 Python 框架

1. Django

  • Cookie：sessionid、csrftoken
  • 路径特征：/admin/ 后台管理页、/static/ 静态资源路径
  • 错误页：Django 调试模式错误信息（暴露代码行号）

2. Flask

  • Cookie：session（经过签名的会话Cookie）
  • 错误页：Flask 调试模式错误页（含交互式控制台）
  • 路径特征：/static/ 静态资源路径

# 🌐 通用 Web 服务器 & 平台

1. Nginx

  • 响应头：Server: nginx/版本号
  • 错误页：403 Forbidden/404 Not Found 标准Nginx页面

2. Apache

  • 响应头：Server: Apache/版本号
  • 特征文件：.htaccess、/server-status

3. Tomcat

  • 响应头：Server: Apache-Coyote/1.1
  • 路径特征：/manager/html 管理后台、/examples/ 示例目录
  • 错误页：Apache Tomcat/版本号 错误信息

4. IIS

  • 响应头：Server: Microsoft-IIS/版本号
  • 特征：.asp/.aspx 后缀、/iisstart.htm 默认页

💡 补充小技巧：

  1. 优先看响应头和Cookie名，是识别框架最快的方式。
  2. 再看URL后缀、路径特征和错误页信息，辅助确认。
  3. 部分框架会通过自定义响应头（如 X-Frame-Options、X-Powered-By）暴露信息。

# AI拓展：

一、安全 & 权限框架

  1. Apache Shiro（高频漏洞框架）
    1）表层易伪造特征
  • Cookie 存在：rememberMe=xxx
  • 响应头：Set-Cookie: rememberMe=deleteMe
  • 常见路由：/login、/logout
    2）底层不可伪造真实指纹（WAF/EDR 精准判定）
  • rememberMe 值 Base64 解码，必含字符串：org.apache.shiro.web.mgt.CookieRememberMeManager
  • 解码开头为 rO0ABX（Java序列化魔数）+ 包含Spring字段 → 判定 Shiro+Spring 漏洞链（CVE-2023-37635）
  • JVM线程堆栈必存在：org.apache.shiro.web.filter.mgt.FilterChainResolver
  • EDR溯源命令：jstack -l 进程PID | grep shiro，命中即为真实Shiro环境
  1. Spring Security
    1）表层易伪造特征
  • 默认安全响应头：X-Frame-Options、X-Content-Type-Options
  • 会话标识：JSESSIONID + XSRF-TOKEN
  • 常见路由：/oauth/token、/login、/logout
    2）底层不可伪造真实指纹
  • 容器固定注册过滤器：springSecurityFilterChain
  • JVM日志固定关键字：UsernamePasswordAuthenticationFilter
  • XSRF-TOKEN解码结构固定：{"token":"xxx","parameterName":"_csrf","headerName":"X-XSRF-TOKEN"}，非此结构即为伪造

二、Java Web 框架

  1. Spring Boot / Spring MVC（本次环境命中框架）
    1）表层易伪造特征
  • 监控端点：/actuator/health、/actuator/env、/actuator/jolokia/
  • 默认错误页：Whitelabel Error Page
  • 接口文档：/swagger/index.html
    2）底层不可伪造真实指纹
  • JVM MBean固定存在：org.springframework.boot:type=Endpoint,name=HealthEndpoint
  • 错误页硬编码字段：<meta name="generator" content="Spring Boot">，无法删除伪造
  • Jolokia端点为Spring Boot微服务专属特征
  1. Struts2
    1）表层易伪造特征
  • URL后缀：.action、.do
  • 响应头：X-Powered-By: Servlet/2.5
    2）底层不可伪造真实指纹
  • 调用栈固定类：com.opensymphony.xwork2.DefaultActionInvocation
  • 注入触发底层：ognl.OgnlValueStack（WAF检测核心关键字）
  1. JSP
    1）表层易伪造特征
  • URL后缀：.jsp、.jspx
  • Cookie：JSESSIONID
    2）底层不可伪造真实指纹
  • Tomcat编译类固定格式：org.apache.jsp.index_jsp
  • 标准JSESSIONID长度固定32位，非此长度为伪造

三、PHP 框架

  1. ThinkPHP
    1）表层易伪造特征
  • 路由格式：/index.php?s=/xxx
  • 响应头：X-Powered-By: ThinkPHP
    2）底层不可伪造真实指纹
  • OPcache常驻类：think\Route
  • 错误页固定路径：thinkphp/library/think/
  1. Laravel
    1）表层易伪造特征
  • Cookie：laravel_session
  • 敏感路径：/storage/logs/laravel.log
    2）底层不可伪造真实指纹
  • Session解密固定字段：_token、previous_url、_flash
  • 日志固定格式：[时间] local.ERROR: xxx

四、.NET 框架
ASP.NET Core
1）表层易伪造特征

  • 服务头：Server: Kestrel
  • 接口文档：/swagger/index.html
    2）底层不可伪造真实指纹
  • 核心请求处理类：Http1Connection
  • Swagger页面固定JS字段：SwaggerUIBundle

五、Python 框架

  1. Django
    1）表层易伪造特征
  • Cookie：sessionid、csrftoken
  • 后台路由：/admin/
    2）底层不可伪造真实指纹
  • Session序列化包含固定对象：django.contrib.auth.models.User
  • 后台路由固定注册方式：admin.site.urls

六、通用Web服务器

  1. Nginx
    表层特征：Server头、标准403/404页面
    底层指纹：403页面标题硬编码不可修改
    高危漏洞：$host变量可控导致SSRF
  2. Apache / Tomcat / IIS
    统一记忆：看响应头、默认管理路径、固定错误页即可快速识别
