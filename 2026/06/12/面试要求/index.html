# 熟悉常见外网打点技术及内外攻击技术

（一）资产测绘与信息搜集（前置打点基础）
外网打点技术，资产测绘与信息搜集
域名资产探测
子域名爆破、DNS 解析查询、旁站 / C 段查询、备案反查、证书透明度 CT 日志爬取、二级 / 三级域名遍历。
端口 & 服务测绘
Masscan、Nmap 全端口扫描，识别公网开放端口：80/443 (Web)、22 (SSH)、3389 (RDP)、3306 (MySQL)、5432 (Postgres)、6379 (Redis)、8080/8443 中间件、VPN (4433/1723)、防火墙管理口。
指纹识别
Web 中间件（Tomcat、WebLogic、JBoss、Nexus、Apache、IIS）、OA/ERP/ 协同系统（泛微、致远、用友、金蝶）、VPN（深信服、天融信、Forti）、路由防火墙、打印机、监控设备、Git/SVN 代码仓库。
OSINT 开源情报
官网公告、招聘、员工邮箱、GitHub/Gitee 泄露源码、网盘泄露配置、第三方平台泄露账号、供应商信息。
（二）Web 服务漏洞打点（最主流外网入口）
通用 Web 漏洞
SQL 注入、XSS 跨站、文件上传、文件包含、目录遍历、SSRF、命令执行、逻辑越权、未授权访问、弱口令后台。
中间件漏洞
WebLogic 反序列化、Tomcat 弱口令 / 后门、Shiro 反序列化、Fastjson、Struts2、Nexus 未授权、Jenkins 命令执行。
专用业务系统漏洞
OA 漏洞、致远 / 泛微任意文件上传 / 命令执行、Exchange ProxyLogon/ProxyShell、Confluence RCE、通达后台漏洞、企业邮箱漏洞。
（三）端口服务弱口令 & 未授权打点
数据库：Redis 未授权访问、MySQL/MSSQL 弱口令、Mongo 未授权、Elasticsearch 未授权漏洞。
远程管理：SSH 弱口令、RDP 爆破、VNC 无密码、WinRM 未授权。
缓存 / 消息队列：Redis 写 Webshell、RabbitMQ 弱口令、Kafka 未授权。
代码仓库：Git/SVN 泄露源码、Gogs/GitLab 弱口令、未授权访问。
（四）边界设备专项打点
SSL VPN/IPSec VPN：弱口令、漏洞 RCE、验证码绕过、短信验证码爆破、Cookie 伪造。
防火墙、负载均衡：管理后台弱口令、命令执行、后台未授权。
路由器、摄像头、打印机：固件漏洞、默认账号、远程命令执行。
（五）钓鱼类外网打点（APT 高频）
鱼叉钓鱼邮件：伪造合作伙伴、法务、招标，宏 Office、恶意 LNK、PDF 漏洞、钓鱼站点。
仿站钓鱼：仿 VPN、OA、企业邮箱登录页窃取账号密码。
社交钓鱼：企业微信、钉钉、LinkedIn 定向发送恶意文件 / 链接。
（六）供应链第三方打点
攻击厂商、外包系统、开源组件投毒，利用企业对外采购的工具、更新服务器实现外网入口。

（一）OWASP Top 10:2025 完整十大漏洞解读
2025 版是 OWASP 最新发布的应用安全风险清单，相较于 2021 版做了大幅调整：将原独立分类的 SSRF 并入访问控制失效，移除旧版 SSRF 分类；新增软件供应链失效、异常情况处理不当两大风险；同时对部分分类名称、覆盖范围进行优化，更贴合云原生、供应链攻击、应用健壮性等当下安全趋势。完整十大项说明如下：
A01:2025 访问控制失效（Broken Access Control）
连续多年位居榜首，是目前 Web 应用中最普遍的安全问题。核心成因是系统未遵循最小权限、默认拒绝原则，权限校验逻辑存在缺陷。典型风险包括水平越权、垂直越权、不安全直接对象引用（IDOR）、后台接口未授权访问、CORS 配置不当、SSRF（2021 版独立项，2025 版归入此类）等。攻击者可绕过权限限制，查看、篡改、删除他人数据或访问管理功能。
A02:2025 安全配置错误（Security Misconfiguration）
排名升至第二位，属于覆盖面极广的配置类漏洞。常见场景有：使用默认账号密码、未关闭目录浏览与调试模式、中间件保留高危默认配置、云资源权限开放、HTTP 安全头缺失、错误页面泄露服务器版本与路径信息等。这类漏洞门槛低，也是自动化扫描工具的重点检测对象，往往是外网渗透的首要突破口。
A03:2025 软件供应链失效（Software Supply Chain Failures）【2025 新增】
在旧版 “老旧脆弱组件” 基础上全面扩展，覆盖整条软件供应链安全。主要风险包括：使用存在高危漏洞的第三方依赖库、组件版本长期不更新、引入不可信开源包、CI/CD 构建链路被篡改、无物料清单（SBOM）管理依赖等。随着应用大量复用开源组件，供应链攻击已成为企业重点防范的风险。
A04:2025 加密机制失效（Cryptographic Failures）
因加密方案设计、使用不当引发的风险。典型问题：用户手机号、密码、交易数据等敏感信息明文存储或传输；使用 MD5、SHA1 等不安全加密算法；密钥硬编码、密钥管理混乱；证书校验不严格等。一旦出现该漏洞，会直接导致核心隐私数据泄露。
A05:2025 注入漏洞（Injection）
经典高危漏洞，当应用直接将用户可控输入拼接到后端解释器执行时触发。包含 SQL 注入、XSS 跨站脚本、命令执行、LDAP 注入、LLM 提示注入等子类。核心根源是没有严格隔离数据与指令边界，该类漏洞危害高、出现频率高，也是 Web 渗透测试的核心挖掘方向。
A06:2025 不安全设计（Insecure Design）
属于架构与业务逻辑层面的底层缺陷，区别于代码实现漏洞。主要是产品设计阶段未做威胁建模、缺少风险预判导致，比如天生存在缺陷的业务流程、无频率限制、并发逻辑漏洞、支付 / 优惠券等业务逻辑漏洞。这类漏洞隐蔽性强，常规扫描工具很难发现，需要人工结合业务流程测试。
A07:2025 身份认证失效（Authentication Failures）
原 “身份标识与认证失效” 简化更名，聚焦账号认证与会话管理问题。常见场景：弱口令、无防护的暴力破解、未部署多因素认证（MFA）、会话令牌可预测、登录会话未正常销毁、密码找回流程存在漏洞等，攻击者容易借此完成账号接管。
A08:2025 软件或数据完整性失效（Software or Data Integrity Failures）
核心是无法保证文件、代码、数据在传输、上传、更新过程中不被篡改。典型风险：文件上传功能未做严格校验，允许上传恶意脚本；系统自动更新、数据反序列化无签名校验；传输数据被篡改等。文件上传、恶意组件植入是该分类下最常见的攻击场景。
A09:2025 安全日志与告警失效（Security Logging and Alerting Failures）
由旧版 “日志与监控失效” 更名而来，强调告警能力的重要性。问题包括：未记录登录、权限变更、数据操作等敏感行为日志；日志泄露明文敏感信息；检测到高危攻击后无实时告警；日志长期不审计。一旦发生安全事件，会导致无法溯源、无法及时应急响应。
A10:2025 异常情况处理不当（Mishandling of Exceptional Conditions）【2025 新增】
应用对非法请求、运行报错、边界异常场景处理不合理。例如：默认报错页面泄露服务器路径、中间件版本、代码详情；异常逻辑下绕过权限校验；非法输入导致服务崩溃等。攻击者可利用异常泄露的信息进一步探测目标、扩大攻击范围。
